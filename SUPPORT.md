# Getting Support

Thank you for using Hyperweaver UI! This document outlines the ways you can get help.

## Available Resources

### Documentation

- **[README](README.md)** — overview, development setup, and build instructions
- Broader platform documentation lives with the Hyperweaver Server / platform docs

### Issues and Bug Reports

- **[GitHub Issues](https://github.com/Makr91/hyperweaver-ui/issues)** — report bugs or request features
- Use the appropriate issue template and search existing issues first to avoid duplicates

### Community Discussion

- **[GitHub Discussions](https://github.com/Makr91/hyperweaver-ui/discussions)** — ask questions and share ideas

## Support Expectations

Hyperweaver UI is maintained with limited development resources:

- **Response time**: usually a few days, but this varies with workload and complexity
- **Priority**: security issues and stability bugs receive the highest priority
- **Feature requests**: evaluated against community need and available time

## Getting Quality Help

### For Bug Reports

1. **Use the bug report template**
2. **Include environment details** (browser + version, OS, Node.js version)
3. **Provide reproduction steps** with expected vs. actual behavior
4. **Include console errors** and relevant logs
5. **Test with the latest version** when possible

### For Questions

1. **Check the README first**
2. **Search existing issues and discussions**
3. **Be specific** about what you're trying to achieve

## Contributing Back

The best way to improve support for everyone:

- **Document solutions** you find to help others
- **Contribute bug fixes** when you solve issues
- **Improve documentation** based on your experience
- **Help answer questions** from other users

See our [Contributing Guide](CONTRIBUTING.md) for more information.
